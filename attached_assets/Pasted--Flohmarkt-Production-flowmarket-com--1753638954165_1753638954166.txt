أريدك أن تُكمل مشروع Flohmarkt وتطلقه في وضع الإنتاج Production على الدومين flowmarket.com مع تفعيل SSL، وتشغيله 24/7. نفّذ المطلوب التالي حرفيًا، وسلّمني كل المخرجات المطلوبة في النهاية:

────────────────────────────────
1) النشر (Deployment)
────────────────────────────────
- انشر الـ Backend (Flask) باستخدام gunicorn:
  gunicorn -w 2 -k gthread -b 0.0.0.0:$PORT app:app
- اجعل التطبيق يعمل على 0.0.0.0 ويقرأ PORT من متغيرات البيئة.
- استخدم منصة تدعم التشغيل الدائم 24/7 (Render / Railway / Replit Deployments (Reserved VM) / VPS).
- فعّل Health Check/Ping للحفاظ على الخدمة مستيقظة.

────────────────────────────────
2) قاعدة البيانات (Persistent DB)
────────────────────────────────
- استخدم SQLite مبدئيًا بمسار دائم (مثل /data/database.db) أو Postgres إذا متاح.
- شغّل سكريبت تهيئة الجداول (init/migration) أوتوماتيكيًا عند أول تشغيل.
- أنشئ حساب الأدمن الافتراضي:
  email: admin@flowmarket.com
  password: admin123
- أنشئ مستخدمًا تجريبيًا عاديًا لاختبار الرفع.

────────────────────────────────
3) خصائص المنصة (الوظائف الأساسية)
────────────────────────────────
- تسجيل / دخول المستخدمين (Register/Login).
- صفحة "إضافة منتج" تسمح برفع الصور (Upload) + السعر + الوصف + اختيار الفئة.
- صفحة "منتجاتي" (My Products) لكل مستخدم لإدارة (تعديل/حذف) منتجاته.
- نظام موافقة الأدمن على المنتجات قبل ظهورها علنًا.
- لوحة إدارة (Admin Panel) على /admin (أو admin.html) تظهر فقط بعد تسجيل الأدمن:
  * إدارة المنتجات (Approve / Reject / Edit / Delete)
  * إدارة المستخدمين (حظر / حذف)
  * إحصائيات: عدد المستخدمين، عدد المنتجات، المنتجات تحت المراجعة
- واجهة الفئات بالصور الاحترافية:
  - سيارات مستعملة → صورة سيارة
  - هواتف محمولة → صورة iPhone
  - إلكترونيات → صورة لابتوب
  - سماعات لاسلكية → صورة سماعات
  - كاميرات احترافية → صورة كاميرا DSLR/Mirrorless
  - أثاث منزلي → صورة صوفا
  - ملابس وإكسسوارات → صورة تيشيرت/إكسسوار
  - فرص عمل → صورة شخص يعمل على مكتب
- دعم اللغة العربية (RTL) في الواجهة بأكملها.
- رفع الصور مع تحديد حجم أقصى، والتحقق من الامتدادات المقبولة.

────────────────────────────────
4) الدومين + SSL
────────────────────────────────
- اربط المشروع بالدومين flowmarket.com.
- أنشئ لي سجلّات DNS المطلوبة، وأرسلها لي بصيغة واضحة لأضعها في لوحة تحكم الدومين:
  * إذا كان الاستضافة تعطي CNAME: 
      CNAME  @   → <your-app-host>
      CNAME  www → <your-app-host>
  * أو إذا كان هناك IP ثابت:
      A      @   → <SERVER_IP>
      CNAME  www → @
- فعّل HTTPS/SSL تلقائيًا (Let's Encrypt أو عبر المنصة).
- فعّل إعادة التوجيه التلقائي من http إلى https، ومن www إلى non-www (أو العكس حسب ما تختاره).
- فعّل HSTS.

────────────────────────────────
5) الأمن والإعدادات
────────────────────────────────
- استخدم ملف .env لإعداد مفاتيح البيئة:
  SECRET_KEY, DB_PATH/DB_URL, ADMIN_EMAIL, ADMIN_PASSWORD, MAX_CONTENT_LENGTH
- فعّل CORS بشكل مضبوط.
- حدّد حجم رفع الصور (MAX_CONTENT_LENGTH).
- سجّل كل الأخطاء (Error Logging) وفعّل مراقبة أساسية (Monitoring).

────────────────────────────────
6) التسليم النهائي (أريد استلام التالي)
────────────────────────────────
- رابط الإنتاج النهائي:
  https://flowmarket.com
- رابط لوحة الإدارة المباشر.
- بيانات الأدمن التي أنشأتها (أخبرني إن غيّرتها).
- ملف README يشرح:
  * كيفية التشغيل محليًا
  * كيف آخذ Backup من قاعدة البيانات
  * كيف أضيف/أعدّل متغيرات البيئة
- قائمة سجلات DNS التي قمت بإنشائها أو المطلوب مني إنشاؤها.
- تأكيد نجاح تفعيل SSL.
- لقطات شاشة (أو GIF قصير) يثبت:
  1) مستخدم عادي يرفع منتج (مع صور)
  2) المنتج يظهر في لوحة الأدمن للموافقة
  3) الأدمن يوافق والمنتج يظهر علنًا في الفئة الصحيحة
- (اختياري) Postman Collection أو curl commands لأهم REST APIs (Register, Login, Add Product, Approve Product).

ابدأ التنفيذ الآن، وعندما تنتهي أعطني كل التفاصيل والروابط المطلوبة أعلاه.
